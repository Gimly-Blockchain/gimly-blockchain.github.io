Decentralized Identifiers

A Decentralized Identifier (DID) is a core component of verifiable data, introducing a unique identifier that operates independently from centralized authorities. The subject associated with the DID can assert control over it using cryptographic proofs. In verifiable data systems, a DID can confirm data's origin and ownership by linking it to the proof's verification method.

DIDs come in a variety of forms. The type, or more accurately, the DID Method, dictates the manner in which CRUD (Create, Read, Update, Delete) operations are implemented for that specific DID. Each DID Method provides a unique set of trade-offs, tailored to suit different use cases. For instance, a did:web can be created without Blockchain, while other DID Methods may depend on it. All variants qualify as DIDs, but they provide different levels of assurances.

DID Creation​

Upon the creation of a DID, it's usually linked with a pair of private and public keys. The public key can be viewed within the DID Document. This setup enables the controller or subject of the DID to produce proofs that anyone possessing the corresponding DID Document for that DID can verify. The method of obtaining the DID Document from a DID is referred to as DID Resolution.

DID Resolution​

A DID Resolver is capable of accepting a DID as input and subsequently retrieving the associated DID Document. This process plays a significant role in the movement of data within verifiable data systems.

DID Document​

Each DID comes with a DID Document that provides details about the DID subject. For did:web, the DID Document is hosted on the website in a specific format. It encompasses vital cryptographic data and the services that the DID offers. This forms the basis for how DIDs can initiate communication with each other.

DID Methods​

A Decentralized Identifier (DID) method is a specification for how a specific DID scheme can be implemented on a certain verifiable data registry, such as a blockchain or distributed ledger. Each DID method specifies the precise syntax and control operations (including create, read, update, deactivate) for its DIDs and DID documents.

The DID method is identified by the scheme-specific string of the DID, which comes after "did:" and before the first colon in the DID. For instance, in the DID "did:example:123456", "example" would be the DID method.

DID methods allow different DIDs to be resolved in different ways, depending on the system where the DIDs are stored. This enables the use of DIDs across various blockchain systems and other types of registries while maintaining a standardized format and operation set.

As of my knowledge cutoff in September 2021, various DID methods had been proposed for different systems, including Bitcoin, Ethereum, Sovrin, and many others. For the most recent and detailed information, please refer to the latest version of the W3C's DID Method Registry.
