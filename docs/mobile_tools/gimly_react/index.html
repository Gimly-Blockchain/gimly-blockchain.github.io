Gimly React Plugin

Gimly React makes it easy to interact with multiple agents in React Applications.

Motivation​

When using gimly in your front end React apps you may need to manage the state of multiple remote and local agents. Gimly React makes it easy to manage agents without needing to write or maintain boilerplate code. It also enables making new features available to front-end stacks without developers needing to implement them manually.

When you add an agent configuration it is persisted to local storage. A randomly generated ID is assigned to each agent.

Install and set up​

yarn add @gimly-community/gimly-react

NOTE: Gimly React depends on the @next versions of @gimly

Installation includes @gimly/core@next and @gimly/remote-client@next. You will NOT need to add additional @gimly dependencies to your app if you are just working with remote agents.

The following sippet is a simplified extract from Gimly Agent Explorer that uses React Query ontop of Gimly React to manage the data layer including caching and global data syncing.

import React from 'react'
import { BrowserRouter, Route } from 'react-router-dom'
import { GimlyProvider } from '@gimly-community/gimly-react'
import { QueryClientProvider, QueryClient } from 'react-query'
import App from '../App'

const queryClient = new QueryClient()

export default = () => (
  <QueryClientProvider client={queryClient}>
    <GimlyProvider>
      <BrowserRouter>
        <Route component={App} />
      </BrowserRouter>
    </GimlyProvider>
  </QueryClientProvider>
)

Create local agent​

Create an agent in your app and export it. You will need to install additional dependencies

yarn add @gimly/did-resolver@next ethr-did-resolver did-resolver web-did-resolver
import { createAgent, IResolver } from '@gimly/core'

import { DIDResolverPlugin } from '@gimly/did-resolver'
import { Resolver } from 'did-resolver'
import { getResolver as ethrDidResolver } from 'ethr-did-resolver'
import { getResolver as webDidResolver } from 'web-did-resolver'

// You will need to get a project ID from infura https://www.infura.io
const INFURA_PROJECT_ID = '<your PROJECT_ID here>'

export const agent = createAgent<IResolver>({
  plugins: [
    new DIDResolverPlugin({
      resolver: new Resolver({
        ...ethrDidResolver({ infuraProjectId: INFURA_PROJECT_ID }),
        ...webDidResolver(),
      }),
    }),
  ],
})

In the provider setup above, add the following to bootstrap the local agent. You can also call addAgent to add while your application is running.

import {agent} from '../gimly'

<GimlyProvider agent={[agent]}>...<GimlyProvider>

useGimly hook​

The primary hook that provides the following API to your app. The below syntax uses React Query to fetch the data and uses the cache key of resolutionResult + agentID to identify the data to your app.

import { useGimly } from '@gimly-community/gimly-react'
import { useQuery } from 'react-query'

export default = () => {
    const { agent } = useGimly<IResolver>()
    const { data } = useQuery(
        ['resolutionResult', { agentId: agent?.context.id }],
        () => agent?.resolveDid({ didUrl: 'did:web:community.gimly.io' })())

    return (
        <div>
            {
                data?.didDocument?.verificationMethod.map((key) => (
                    <div>{JSON.stringify(key)}</div>
                )
            }
        <div>
    )
}

If you are not using React Query you can just call agent?.resolveDid() and manage the data like any async data source.

API​

agent​

The current active agent object. Call agent methods as normal:

agent[METHOD]

agents​

A list of all configured agents.

activeAgentId​

The ID of the currently active agent.

setActiveAgentId​

Set the current active agent by ID

addAgent​

Add a local agent. Create a local agent as per example in Create local agent section.

import { agent } from '../gimly'
import { useGimly } from '@gimly-community/gimly-react'

// Inside a function component

const { addAgent } = useGimly()

const addLocalAgent = () => {
  addAgent(agent)
}

removeAgent​

Remove an agent by ID.

addAgentConfig​

Add a remote agent configuration.

import { useGimly } from '@gimly-community/gimly-react'

// Inside a function component
const { addAgentConfig } = useGimly()

const newAgentConfig = () => {
  addAgentConfig({
    context: { name: 'Agent Name', schema: schemaUrl },
    remoteAgents: [
      {
        url: agentUrl,
        enabledMethods: Object.keys(schema['x-methods']),
        token: apiKey,
      },
    ],
  })
}

updateAgentConfig​

Update the configuration of an agent.

getAgentConfig​

Get the current configuration for an agent.

getAgent​

Get an agent by ID.
